# Protocol Services

Protocol services built on the transport layer providing protocol-specific functionality.

## Architecture Overview

Protocol services integrate with transports to handle protocol-specific logic:

```
┌─────────────────────────────────────────┐
│ Protocol Service                        │  ← Protocol-specific implementation
│ (with optional service middleware)      │
├─────────────────────────────────────────┤
│ Transport Layer                         │  ← HTTP / WebSocket
│ (with optional transport middleware)    │
├─────────────────────────────────────────┤
│ Network                                 │  ← Network communication
└─────────────────────────────────────────┘
```

**Request Flow:** Protocol Service → [Service Middleware] → Transport → [Transport Middleware] → Network

**Response Flow:** Network → [Transport Middleware] → Transport → [Service Middleware] → Protocol Service

> **Note:** Middleware is optional and composable. It can be applied at the service level (protocol-specific) or transport level (transport-specific), or both.
