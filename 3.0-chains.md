# Building Blockchain RPC Clients

This document demonstrates how to build blockchain RPC clients using the sb-stack components. It shows how to assemble the JSON-RPC service, organize methods with namespaces, add batch processing, WebSocket subscriptions, and transaction signing.

## Architecture Overview

The sb-stack provides a layered architecture for building blockchain clients:

```
┌─────────────────────────────────────┐
│     Blockchain Client               │  ← High-level blockchain API
│  (Your application logic)           │
├─────────────────────────────────────┤
│   JSON-RPC Service   BatchService   │  ← JSON-RPC protocol layer
│  (single requests)  (batch requests)│
│   + Signer          + PubSubService │
├─────────────────────────────────────┤
│     Service Level Middleware        │  ← Logging, retry, telemetry
├─────────────────────────────────────┤
│   HTTP Transport    WebSocket       │  ← Transport implementations
│  (HTTP/1.1 or H2)  (subscription)   │
├─────────────────────────────────────┤
│   Transport Level Middleware        │  ← Auth, rate limiting, etc.
├─────────────────────────────────────┤
│        Transport Service            │  ← Core transport abstraction
└─────────────────────────────────────┘
```
